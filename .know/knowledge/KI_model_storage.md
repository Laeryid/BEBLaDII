# KI: Model Storage & Lifecycle

## Что это
Рекомендации и правила хранения весов моделей (внешних и собственных), управления кэшем и стратегии работы в разных окружениях (Local vs Kaggle).

## Локальное хранение (Windows)
*   **Путь к кэшу HuggingFace**: `C:\Users\hp\.cache\huggingface\hub` (стандартное поведение библиотек `transformers` и `huggingface_hub`).
*   **Управление кэшем**: 
    *   Регулируется переменной окружения `HF_HOME`.
    *   Анализ места: `huggingface-cli cache scan`.
*   **Правило**: Не хранить внешние (базовые) модели в папке проекта.

## Стратегия версионирования (Git)
*   **Git LFS (Large File Storage)**: Обязателен для всех файлов `.pt` и `.bin` внутри `storage/`. 
*   **Чекпоинты**:
    *   `storage/components/` — веса отдельных модулей (проекторы, ученик). Сохраняются через `ComponentRegistry`.
    *   `storage/experiments/` — комплексные состояния фаз вместе с оптимизатором. Сохраняются через `ExperimentTracker`.
*   **Конфиги**: `.json` файлы конфигураций ОБЯЗАТЕЛЬНО хранятся в обычном Git (без LFS) для отслеживания изменений архитектуры.

## Работа на Kaggle
| Ресурс | Стратегия | Механизм |
|---|---|---|
| **Base Models** (ModernBERT, DeepSeek) | Динамическая загрузка | Прямо в ноутбуке с HF (рекомендуется `hf_transfer`) |
| **Custom Weights** (Наши наработки) | Постоянное хранение | Private Kaggle Dataset |

**Рекомендация по квотам:** Не тратить GPU-время на долгую скачку больших весов. Самые тяжелые компоненты (ученик после DUS) лучше заливать как Dataset.

## Юридические аспекты и Лицензии
*   **Apache 2.0**: Основная лицензия проекта. Позволяет публичное распространение кода и весов.
*   **Атрибуция**: При публикации весов, полученных методом дистилляции (например, из моделей DeepSeek/MIT), в README должен быть указан первоисточник.
