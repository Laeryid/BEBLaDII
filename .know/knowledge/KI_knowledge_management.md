# KI: KI_base (Knowledge Management System)

## Что это
Автоматизированная экосистема ведения базы знаний проекта (Knowledge Items), построенная на принципе самодокументирования и аудита. Позволяет ИИ-агентам поддерживать актуальность технического контекста.

## Ключевые компоненты
| Класс / Функция | Файл | Назначение |
|---|---|---|
| **KnowledgeManager** | `.know/scripts/knowledge_mcp.py` | MCP-сервер, предоставляющий интерфейс для работы с документацией в изолированной среде. |
| **Knowledge Engine** | `.know/scripts/knowledge_engine.py` | Ядро системы: хеширование файлов, отслеживание изменений и расчет зависимостей (Staleness). |
| **Audit Coverage** | `.know/scripts/audit_coverage.py` | Инструмент построения матрицы покрытия, оценки плотности (Density) и сложности (Complexity) KI. |
| **Document Config** | `.know/doc_config.json` | Манифест системы: настройки трекинга, список KI и их связи с кодом. |
| **DIR Index Gen** | `.know/scripts/generate_dir_index.py` | Генерация компактного дерева директорий проекта для контекста агента. |
| **KI Registrator** | `.know/scripts/add_ki_to_config.py` | CLI-инструмент для автоматического добавления новых KI в `doc_config.json`. |
| **Sync Tools** | `.know/scripts/sync_agents_md.py` | Скрипты синхронизации AGENTS.md. |

## Неочевидные детали
- **Jailhouse (Песочница)**: MCP-инструменты `read_know_file` и аналогичные жестко ограничены директорией `.know`. Попытка выйти за её пределы через относительные пути (`../`) блокируется функцией `validate_path`.
- **Security Block-list**: Через MCP-сервер запрещено изменять исполняемые файлы (`.py`, `.bat` и т.д.) во избежание RCE. Редактирование разрешено только для безопасных форматов (Markdown, JSON).
- **Density Threshold**: Установлен порог 50.0 B/KB. Если на 1 КБ исходного кода в KI-файле меньше 50 байт описания, статус помечается как Low Density (❄️).
- **Complexity Guard**: Если один KI-файл через `depends_on` отслеживает более 10 файлов, аудит выставляет предупреждение Complexity Warning (🔥), рекомендуя разделение документа на более узкие темы.
- **State Capture**: Файл `doc_state.json` хранит SHA256-хеши всех отслеживаемых файлов для мгновенного определения изменений.

## Типичные ошибки
- **Jail Breach**: Ошибка PermissionError при попытке доступа к коду проекта через инструменты KnowledgeManager. Для кода используйте стандартные инструменты просмотра (view_file).
- **Manual Edit Desync**: Прямое редактирование `doc_state.json` не рекомендуется; используйте `save_state` после внесения изменений в KI.
- **Dependency Missing**: Если новый модуль не добавлен в `tracked_modules` внутри `doc_config.json`, он не попадет в аудит.
