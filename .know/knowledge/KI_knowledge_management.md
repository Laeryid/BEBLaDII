# KI: KI_base (Система управления знаниями)

## Что это
Автоматизированная экосистема ведения базы знаний проекта (Knowledge Items), построенная на принципе самодокументирования и аудита. Позволяет ИИ-агентам поддерживать актуальность технического контекста. Все ресурсы системы сосредоточены в `knowledge_root` (директория `.know`).

## Ключевые компоненты
| Компонент | Путь (отн. `knowledge_root`) | Назначение |
|---|---|---|
| **KnowledgeManager** | `scripts/knowledge_mcp.py` | MCP-сервер, предоставляющий интерфейс для работы с документацией. |
| **Knowledge Engine** | `scripts/knowledge_engine.py` | Ядро системы: хеширование файлов, отслеживание изменений и Staleness. |
| **Audit Coverage** | `scripts/audit_coverage.py` | Инструмент построения матрицы покрытия, оценки плотности и сложности. |
| **Workflows** | `workflows/` | Набор инструкций (слэш-команд) для выполнения типовых задач агентом. |
| **ADR (Decisions)** | `decisions/` | Журнал принятия архитектурных решений (Architecture Decision Records). |
| **Infra Tests** | `tests/` | Набор тестов для проверки безопасности MCP и логики хеширования. |
| **Document Config** | `doc_config.json` | Манифест системы: настройки трекинга, список KI и их связи с кодом. |
| **DIR Index Gen** | `scripts/generate_dir_index.py` | Генерация компактного дерева директорий проекта. |
| **Sync Tools** | `scripts/sync_agents_md.py` | Скрипты синхронизации `AGENTS.md`. |

## Основные механизмы

### Воркфлоу (Workflows)
Воркфлоу — это Markdown-инструкции в папке `workflows/`, которые агент может вызывать через `/команду`. Каждый файл должен содержать YAML-блок `description` в начале для корректной индексации системой.

### Журнал решений (ADR)
Архитектурные решения фиксируются в `decisions/` в формате `<ID>_<topic>.md`. ADR строго привязаны ко времени и не редактируются после принятия; изменения фиксируются в новых документах.

### Тестирование инфраструктуры
В папке `tests/` находятся тесты для ядра системы знаний. Они проверяют:
- Изоляцию MCP-сервера в песочнице (Jailhouse).
- Корректность расчета хешей и матрицы покрытия.
- Успешную генерацию индексов директорий.

## Неочевидные детали
- **Jailhouse (Песочница)**: MCP-инструменты `read_know_file` и аналогичные ограничены директорией `knowledge_root`. Попытка выхода через `../` блокируется.
- **Security Block-list**: Через MCP-сервер запрещено изменять исполняемые файлы (`.py`, `.bat`). Редактирование разрешено только для Markdown и JSON.
- **Density Threshold**: Порог 50.0 B/KB. Если плотность описания ниже, статус помечается как Low Density (❄️).
- **Complexity Guard**: Если один KI отслеживает более 10 файлов, выставляется Complexity Warning (🔥).
- **State Capture**: Файл `doc_state.json` хранит SHA256-хеши для определения изменений.

## Типичные ошибки
- **Jail Breach**: Ошибка PermissionError при попытке доступа к коду через KnowledgeManager. Используйте `view_file` для кода вне `knowledge_root`.
- **Manual Edit Desync**: Не рекомендуется править `doc_state.json` вручную; используйте `save_state`.
- **Dependency Missing**: Если модуль не добавлен в `doc_config.json`, он игнорируется аудитом.
