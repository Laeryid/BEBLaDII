# KI: Architecture & Training Phases

## Что это
Высокоуровневое описание архитектуры BEBLaDII (Reasoning Latent Diffusion) и стратегии итеративного обучения компонентов. Система реализует принцип разделения процессов логического анализа (System 2, латентная диффузия) и лингвистической генерации (System 1, декодер).

## Ключевые компоненты
| Компонент | Класс / Путь | Назначение |
|---|---|---|
| **System 2 Engine** | `DUSModel` (40 layers) | Ядро диффузии на базе ModernBERT, выполняющее итеративное уточнение смыслов. |
| **Reasoning Distiller**| `ReasoningDistiller` | Класс-оркестратор дистилляции знаний из DeepSeek-R1 в ModernBERT. |
| **Input Projector** | `distiller.input_projector` | Проектирует эмбеддинги Qwen (3584) в пространство ModernBERT (1024). |
| **Feature Projectors**| `nn.ModuleDict` | Набор проекторов для выравнивания промежуточных слоев [14, 21, 28] учителя и [20, 30, 40] latentBERT. |
| **Prompt CA Layers**| `ModuleList` в составе DUS | 3 слоя Cross-Attention для внедрения пользовательского промпта. |
| **RAG CA Layers** | `ModuleList` в составе DUS | 3 слоя Cross-Attention для внедрения контекста базы знаний (RAG). |
| **System 1 (Voice)** | `DeepSeek-R1-Distill-Qwen`| Замороженный декодер, определяющий целевое семантическое пространство. |

## Реализация фаз обучения
Процесс обучения первой фазы (Alignment) реализован в `experiments/train_phase1_kaggle.py` и разделен на два этапа:

1. **Stage 1: Awakening**
   - **Цель**: Инициализация латентного пространства ученика.
   - **Логика**: Модель учится воспринимать проекции входов учителя через `input_projector`.
   - **Обучаемые веса**: `latentBERT` + `input_projector`.

2. **Stage 2: Reasoning**
   - **Цель**: Глубокая дистилляция "хода мыслей".
   - **Логика**: Обучение `feature_projectors` передавать семантические нюансы из глубоких слоев учителя в соответствующие слои ученика. Доступна загрузка словарей весов с предыдущих стадий через `CUSTOM_STUDENT_WEIGHTS_PATH`.
   - **Обучаемые веса**: `latentBERT` + `input_projector` + `feature_projectors`.

## Механизм обусловливания (Conditioning)
Мы используем парадигму **Conditional Latent Diffusion**. Вместо конкатенации входного промпта со стартовым шумом (что приводит к "размытию" инструкций на глубине 40 слоёв), система внедряет внешние сигналы через разреженные слои Cross-Attention (CA):
- **Промпт**: подается через 3 CA-слоя, расположенных после full-attention блоков (в начале, середине и конце модели). 
- **RAG-контекст**: подается через 3 отдельных CA-слоя (после других full-attention блоков). Используется весовой параметр `gamma` для балансировки влияния RAG.
Такое разделение позволяет независимо балансировать внимание к инструкции (команде) и фактологии (справке), а также открывает путь для чистого Classifier-Free Guidance.

## Неочевидные детали
- **DUS (Depth Up-Scaling)**: latentBERT инициализируется как 40-слойная модель из 24-слойного базового ModernBERT-large через механизм дублирования средних блоков.
- **Оптимизация памяти (OOM & GC)**: Для длинных последовательностей включен `Gradient Checkpointing` студента, `use_cache=False`, и механизмы перехвата ошибок Out of Memory с очисткой кэша PyTorch: `torch.cuda.empty_cache()` и ручным сбросом `optimizer.zero_grad()`.
- **Лосс**: Используется `DistillationLoss`, который минимизирует MSE и Cosine Similarity (Mask-Aware) между спроецированными состояниями latentBERT и таргетами учителя.

## Типичные ошибки
- **Mismatch в размерностях**: Токены Qwen2.5 имеют dim 3584, ModernBERT — 1024. Прямое сравнение невозможно без проектора.
- **NaN Loss**: Часто возникает в начале Stage 2, если пропустить Stage 1. Отрабатывается скриптом через `NaN loss detected... Skipping batch`.
- **Заморозка учителя**: Учитель должен быть в режиме `.eval()` и с `requires_grad=False` для экономии VRAM.
- **weights_map**: При запуске обучения проверяй логи `build_weights_map()` — статус `[found]`/`[random init]` покажет, загружены ли сохранённые веса или инициализированы заново.
